## 5. 申請後の対応

### 検証環境が届いたら

国土交通省から以下が届きます。

1. **クライアントIDの発行通知**（検証環境用）
2. **DIPS2.0 API 設定通知書**（検証環境用）

設定通知書には、検証環境で使うべきURL、確認ポイントなどが記載されています。

### 検証環境で確認すること

| 確認項目 | 内容 |
|---------|------|
| 認可リクエスト | ログイン後、redirect_uri に正しくリダイレクトされること |
| state 検証 | リクエストとレスポンスの state が一致すること |
| トークン取得 | 認可コードで access_token が取得できること |
| id_token 検証 | id_token が正常に検証できること |
| 属性取得 | access_token で UserInfo が取得できること |
| 飛行計画検索 | リクエスト/レスポンスが仕様通りであること |
| 飛行禁止エリア検索 | リクエスト/レスポンスが仕様通りであること |
| 飛行計画通報 | リクエスト/レスポンスが仕様通りであること |

### 検証完了後

すべての確認が完了したら、国土交通省に「検証環境動作確認完了報告」をメールで送信します。すると本番環境用の client_id が発行されます。

## 6. 注意事項

- **本番環境での負荷試験・異常系テストは禁止**されています
- 検証環境で十分にテストしてから本番に進んでください
- client_secret は絶対にフロントエンド（ブラウザ側）に公開しないでください
- DIPSアカウント（ID/パスワード）はアプリ内に保存しないでください
